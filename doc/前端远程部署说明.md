# 前端远程部署说明

## 1. 当前方案

当前前端已经补成独立远端部署栈：

- 目标机目录：`/opt/school-erp/stacks/erp-frontend`
- 目标机容器：`school-erp-frontend-frontend-1`
- 目标机监听：`127.0.0.1:18200`
- 跳板机转发：`0.0.0.0:90 -> 目标机 127.0.0.1:18200`

前端容器内部使用 `nginx`：

- 直接托管构建后的静态资源
- 对 `/api` 反向代理到目标机现有网关 `127.0.0.1:18080`
- 对 Vue 单页路由做 `index.html` 回退

因此浏览器访问前端时，不需要再单独配置 `VITE_API_BASE_URL`，前端直接走同源 `/api` 即可。

## 2. 仓库内相关文件

- 前端远端栈：`infra/remote-frontend/docker-compose.yml`
- 前端运行镜像：`infra/remote-frontend/Dockerfile`
- Nginx 配置：`infra/remote-frontend/nginx.default.conf`
- 部署脚本：`scripts/deploy-remote-frontend.py`
- PowerShell 包装：`scripts/deploy-remote-frontend.ps1`

## 3. 使用方式

默认推荐：

```powershell
.\scripts\deploy-remote-frontend.ps1 -JumpPrivateKeyFile C:\Users\fan\.ssh\id_rsa
```

该脚本会执行：

1. 本地构建 `frontend/dist`
2. 经跳板机上传到目标机 `/opt/school-erp/stacks/erp-frontend`
3. 在目标机执行 `docker compose up -d --build`
4. 在跳板机创建并启动 `bridgeability-frontend-port90.service`

## 4. 当前验证结果

已经确认：

- 目标机 `127.0.0.1:18200` 可返回前端首页
- 跳板机 `bridgeability-frontend-port90.service` 已启动
- 跳板机本机访问 `http://127.0.0.1:90/` 可返回前端首页

## 5. 当前剩余限制

当前从外部访问 `http://8.148.181.9:90/` 仍然超时。

这一步已经不是仓库脚本、Docker 或 Linux 本机防火墙问题，更像是跳板机云侧安全组没有放行 TCP `90` 入站。

如果要真正从公网访问，需要额外完成：

- 在云控制台给跳板机放行入站 TCP `90`

放行后，当前已经运行的 systemd 转发服务可直接接管，不需要再改前端容器本身。
