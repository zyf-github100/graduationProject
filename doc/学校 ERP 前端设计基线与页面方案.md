# 学校 ERP 前端设计基线与页面方案

## 1. 设计思路

本项目不是营销官网，也不是展示型 Dashboard，而是面向学校管理员、教务老师、班主任、任课教师、财务人员长期使用的 ERP 管理系统。界面设计必须首先服务于业务处理效率、状态判断准确性、信息检索清晰度和长期办公的视觉稳定性。

结合《系统总体架构设计说明书》《接口设计规范说明书》《接口清单明细版》和《正式版目录结构与甘特图排期》中的业务边界，本前端设计以以下原则为核心：

1. 优先解决高频后台场景：列表、筛选、表单、审批、详情、数据对账、异常提示。
2. 用统一的布局和组件规则承接认证权限、基础数据、教务、审批、收费、消息六大模块。
3. 保持中高信息密度，但通过层级、分组、留白和状态色控制可读性，避免拥挤。
4. 以桌面端管理后台为主场景，响应式只做自然收缩，不牺牲主工作区效率。
5. 页面风格克制、稳定、可信，不使用炫技式视觉语言，不制造 AI demo 感。

## 2. 页面结构

### 2.1 一级信息架构

建议管理端采用固定左侧主导航 + 顶部全局栏 + 主工作区 + 局部工具栏的后台结构：

1. 左侧主导航：承载一级模块与二级菜单。
2. 顶部全局栏：承载系统标题、当前学期、校区切换、全局搜索、消息、用户信息。
3. 页面头部：承载页面标题、面包屑、主操作、状态摘要。
4. 内容区：按业务场景切换为筛选区、表格区、详情区、表单区、时间线区等。
5. 右侧补充区：仅在审批、详情、对账、课表冲突等需要上下文时出现。

### 2.2 核心页面模板

1. 登录页：品牌识别、登录表单、公告/系统说明、环境信息。
2. 系统首页：待办、预警、今日教学安排、收费与审批摘要、常用入口。
3. 列表页：筛选区 + 工具栏 + 数据表格 + 分页。
4. 详情页：主信息卡 + 状态流 + 分组详情 + 操作记录。
5. 新建/编辑页：分区表单 + 吸顶操作栏 + 校验提示。
6. 审批页：申请信息 + 流转记录 + 审批动作区 + 关联业务快照。
7. 课程/成绩/学生管理页：以筛选、表格、批量操作和状态管理为主。
8. 个人中心/设置页：账号信息、密码安全、通知偏好、界面偏好。

## 3. 组件复用方案

### 3.1 布局复用

1. `AppShell`：系统级外壳，统一左侧导航、顶部栏、内容容器。
2. `PageSection`：页面分区容器，统一标题、操作区和内容间距。
3. `DataToolbar`：列表工具栏，统一新建、导出、批量操作、刷新。
4. `FilterPanel`：筛选容器，统一查询项布局、展开收起、重置查询。

### 3.2 业务组件复用

1. `StatusBadge`：统一审批、账单、成绩、考勤等状态颜色和文案。
2. `MetricPanel`：首页统计模块，统一标题、主数值、趋势、说明。
3. `RecordTable`：带斑马纹、选中态、固定操作列的数据表格基类。
4. `DetailGroup`：详情页信息分组，统一 label/value 栅格与敏感字段展示。
5. `FormSection`：表单分组容器，统一标题、说明和字段间距。
6. `ProcessTimeline`：审批与操作记录时间线。
7. `EmptyPlaceholder`：空状态组件，统一图形、说明、操作建议。

### 3.3 页面生成策略

1. 列表类页面优先由 `FilterPanel + DataToolbar + RecordTable + Pagination` 组合生成。
2. 详情类页面优先由 `SummaryHeader + DetailGroup + ProcessTimeline` 组合生成。
3. 表单类页面优先由 `FormSection + StickyActionBar` 组合生成。
4. 审批类页面优先由 `ApplicationSummary + ApprovalActionPanel + ProcessTimeline` 组合生成。

## 4. 为什么这个方案适合学校 ERP

1. 学校 ERP 的核心不是“展示”，而是“管理”和“留痕”，所以模板优先支持列表、表单、审批、详情、快照和日志。
2. 六大服务模块虽然业务不同，但都依赖统一的状态表达、权限控制、筛选检索和数据展示方式，这套设计可以跨模块复用。
3. 学校管理场景涉及学生、教师、课程、课表、成绩、账单、流程等大量结构化数据，中高密度布局比大卡片式布局更适合长期办公。
4. 财务、教务、审批都要求正式和可信，克制配色、清晰分组、稳定边框和保守动效比炫酷视觉更符合真实系统预期。

## 5. 设计基线

### 5.1 产品视觉气质

正式、理性、稳定、可信、克制。视觉上强调秩序感和操作确定性，而不是情绪表达。页面给人的感受应接近“学校信息化平台”和“政府/事业单位业务系统”的专业度，但避免传统系统的粗糙和压迫感。

### 5.2 设计关键词

1. 清晰
2. 统一
3. 稳定
4. 可信
5. 高效
6. 克制
7. 可维护
8. 中高密度

### 5.3 配色策略

采用“中性色为主、单主色驱动、功能色辅助”的策略：

1. 主色使用冷静的学院蓝，表达正式与秩序。
2. 背景以低对比灰白层级区分页面、容器、输入区域。
3. 文本使用高可读性中性灰黑，不用纯黑。
4. 状态色严格服务于业务状态，不做装饰用途。
5. 控制页面中同时出现的强调色数量，避免信息竞争。

建议色彩角色：

1. Primary：导航当前态、主按钮、链接、重点数据。
2. Neutral：背景、分割线、正文、说明文字、禁用态。
3. Success：通过、启用、支付成功、已发布。
4. Warning：待处理、即将到期、风险提醒。
5. Danger：驳回、逾期、欠费、删除、错误。
6. Info：系统提示、说明、处理中。

### 5.4 字体层级

系统默认采用适合中文后台的无衬线字体栈，优先 `PingFang SC`、`Microsoft YaHei`、`Noto Sans SC`、`sans-serif`。

层级建议：

1. 页面标题：24px / 600
2. 区块标题：18px / 600
3. 表格标题与卡片标题：16px / 600
4. 正文主信息：14px / 400-500
5. 表单标签与表格正文：13px / 400
6. 辅助说明：12px / 400
7. 指标数字：24-32px / 600

行高建议：

1. 标题行高 1.4
2. 正文行高 1.6
3. 表格行高 1.5

### 5.5 Spacing System

采用 4px 基础单位，形成稳定的后台间距系统：

1. 4：细微内边距、标签间距
2. 8：控件内边距、小型元素间距
3. 12：表单项间距、行内组合间距
4. 16：常规区块内边距
5. 20：工具栏和卡片内部留白
6. 24：页面区块间距
7. 32：大分组间距
8. 40：页面首屏和大模块留白

### 5.6 Border / Radius / Shadow 规则

1. Border：统一使用 1px 实线边框，颜色低对比，不使用重描边。
2. Radius：以 6px 和 8px 为主，不使用过圆角。
3. Shadow：仅用于弹层、下拉、浮层和少量重点面板，阴影浅且克制。
4. 卡片和区域分组优先依靠背景层级与边框区分，而不是厚重阴影。

建议：

1. 输入框、按钮、标签：6px
2. 卡片、筛选区、详情区：8px
3. Drawer、Modal：10px
4. 页面主容器默认无明显投影

### 5.7 页面布局原则

1. 页面采用 24 栅格思路，但实际后台中以左右分栏和纵向分组为主。
2. 页面宽度优先自适应全宽，避免被大容器过度限制。
3. 页面头部、筛选区、数据区、详情区之间保持固定节奏。
4. 列表页优先使用“筛选区在上、表格在下”的结构。
5. 详情页优先使用“主信息概览在上、分组详情在下”的结构。
6. 表单页优先按业务语义分组，而不是机械地平均分栏。
7. 响应式主要处理筛选项换行、侧边栏收缩、详情区堆叠，不改变核心交互模型。

### 5.8 导航系统建议

建议采用“两级导航为主、三级入口谨慎使用”的策略：

1. 一级导航：认证权限、基础数据、教务管理、审批中心、收费管理、消息中心、系统设置。
2. 二级导航：对应具体业务，如学生档案、教师档案、成绩管理、账单管理、待办中心。
3. 顶部面包屑：负责帮助用户理解当前位置。
4. 常用入口：首页快捷入口和最近访问，提高高频任务效率。
5. 全局搜索：预留教师、学生、单号、账单号等统一搜索入口。

### 5.9 表格、表单、详情、筛选区统一设计方式

#### 表格

1. 表头背景轻微区分，字体加粗但不过重。
2. 行高控制在 40-44px，保证密度与可读性平衡。
3. 表格支持固定列、批量选择、状态标签、行级操作。
4. 敏感字段采用脱敏或权限控制展示。
5. 操作列固定在右侧，避免横向滚动后丢失动作。

#### 表单

1. 表单宽度不宜过满，字段按逻辑分组。
2. 标签对齐规则固定，不混用左对齐和顶对齐。
3. 必填、帮助信息、校验反馈采用统一规范。
4. 高风险操作提供二次确认与说明。
5. 底部操作栏保持可见或吸顶，避免长表单来回滚动。

#### 详情

1. 顶部展示编号、状态、主操作和关键摘要。
2. 中部按“基础信息 / 业务信息 / 附件 / 留痕记录”分组。
3. 变更记录、审批记录和操作日志采用时间线或日志表格统一呈现。
4. 详情值优先可扫描，不堆砌装饰性模块。

#### 筛选区

1. 默认展示高频筛选项，其余放入展开区域。
2. 筛选区与表格区保持稳定距离。
3. 查询、重置、展开、导出的位置固定。
4. 支持学期、校区、班级、状态、时间范围等 ERP 常见维度。

### 6.4 Layout Rules

1. 顶部全局栏高度建议 56px。
2. 左侧导航展开宽度建议 232px，收起宽度建议 72px。
3. 页面内容左右边距建议 24px。
4. 页面头部与内容区间距建议 20-24px。
5. 筛选区、详情区、卡片区采用统一容器宽度与内边距。

### 6.5 Button Variants

1. Primary：提交、保存、确认、发布。
2. Secondary：普通操作、返回、查看。
3. Outline：次级强调、筛选展开、工具类动作。
4. Text：低干扰操作，如详情跳转、辅助链接。
5. Danger：删除、驳回、关闭、作废。

规则：

1. 主页面同一区域只保留一个主按钮。
2. 高风险按钮必须颜色和确认流程同时区分。
3. 按钮尺寸控制在中等高度，避免后台界面显得稚嫩。

### 6.6 Form Controls Styles

1. 输入框背景纯白，边框低对比。
2. 聚焦态使用主色边框和轻微外圈。
3. 禁用态弱化文本和背景，不改变布局。
4. 错误态以边框和说明文结合，不只靠颜色。
5. 下拉、日期、选择器与输入框高度统一。

### 6.7 Table Styles

1. 表头背景使用轻灰蓝中性色。
2. 行 hover 态轻微高亮，不使用强对比。
3. 选中行用低饱和主色底。
4. 表格内部操作按钮采用 text 或 link 风格，防止噪音过大。
5. 大表格支持工具栏、列宽控制、分页和批量操作。

### 6.8 Card Styles

1. 卡片只用于分组和摘要，不做视觉主角。
2. 默认白底、细边框、8px 圆角。
3. 指标卡内部重点突出数字和状态，而非装饰。
4. 同页卡片结构统一，避免每张卡风格不同。

### 6.9 Modal / Drawer / Tabs / Tag / Badge / Empty State / Alert

#### Modal

1. 用于高专注确认和简短编辑。
2. 宽度按内容等级分为小、中、大三档。
3. 标题、说明、正文、底部操作间距统一。

#### Drawer

1. 用于查看详情、补充信息、复杂侧边编辑。
2. 适合不打断列表页上下文的场景。

#### Tabs

1. 用于详情页子维度切换，如“基本信息 / 附件 / 留痕”。
2. 样式稳重，避免花哨切换动效。

#### Tag / Badge

1. 统一承载状态、类别、角色、来源。
2. 使用低饱和底色和中深色文字。
3. 状态色与业务状态映射必须稳定。

#### Empty State

1. 使用简洁线性图形或中性占位图。
2. 文案直接说明原因与下一步动作。
3. 不做夸张插画。

#### Alert

1. 信息提示、风险提示、成功反馈采用统一横幅样式。
2. Alert 服务于说明和风险提醒，不用于堆砌视觉强调。

## 8. 实现约束

1. 所有页面延续同一套色彩、排版、间距和容器规则。
2. 优先复用基础布局、表格、表单、状态、详情组件。
3. 不为单页需求临时新增一套视觉语言。
4. 响应式只做必要收缩和换行，桌面端始终优先。
5. 新业务模块必须先映射到现有页面模板中，再扩展局部组件。

## 9. 自检清单

1. 这个页面是否像真实学校正在使用的正式系统？
2. 是否适合长时间办公使用？
3. 信息层级是否清晰？
4. 表格、表单、筛选是否足够稳定和易用？
5. 是否避免了 AI demo 感和廉价模板感？

如果任一项答案是否定的，优先调整布局秩序、层级表达、组件复用和状态表达，而不是额外增加视觉装饰。
